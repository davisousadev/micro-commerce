# Regras e diretrize do projeto

## Sobre
Projeto de implementação de um micro-ecommerce com os seguintes serviços:
- Catálogo de produtos
- Pedidos
- Notificações

## Regras Gerais
- Cada serviço deve ser implementado em um repositório separado.
- Sempre responda em português do Brasil.
- Não crie arquivos sem que seja solicitado.
- Só crie código após uma conversa e alinhamento das tecnologias e ferramentas a serem utilizadas.
- Não utilize ferramentas que não estejam explicitamente autorizadas.
- Se não tiver certeza de algo, pergunte.

## Arquitetura de pastas
- api-getway: Gateway de API responsável por receber as requisições e encaminhá-las para os serviços específicos.
- products-service: Serviço responsável pela gestão de produtos.
- orders-service: Serviço responsável pela gestão de pedidos.
- notifications-service: Serviço responsável pelo envio de notificações.
